# 자기소개서 작성 가이드라인 (합격 자소서 패턴 기반)

## 핵심 원칙

**"합격하는 자기소개서"는 "잘 쓴 연구자 소개문"과 다르다.**

- ❌ 논문/성과 중심 나열
- ✅ 사람 중심 서술 (성향 → 관심 → 경험 → 직무 연결)

---

## 1. 합격 자소서의 전형적 구조

### 자기소개/자유형 질문

```
1. 나의 성향·가치관·관심사 (사람 중심 도입)
2. 해당 성향이 직무/분야에 자연스럽게 연결된 이유
3. 이를 보여주는 대표 경험 1~2개 ← 여기서만 연구/성과 사용
4. 그 경험이 사고방식/일하는 방식에 준 영향
5. 회사/직무와의 연결 및 마무리
```

### 지원동기 질문

```
1. 회사/직무에 관심 갖게 된 계기
2. 나의 경험/역량과 연결점
3. 구체적 기여 방향
```

### 직무역량 질문

```
1. 핵심 역량 제시
2. 역량을 보여주는 경험 (문제-해결-결과)
3. 해당 역량의 직무 적용 방향
```

---

## 2. 질문 유형별 허용/금지 사항

| 질문 유형 | ✅ 허용 | ⚠️ 보조만 | ❌ 금지 |
|----------|--------|----------|--------|
| 자기소개 | 성향, 가치관, 태도 | 실적 1~2개 | 성과 나열, 논문 리스트 |
| 지원동기 | 회사 관심 이유, 적합성 | 관련 경험 | 일반적 칭찬, 뻔한 표현 |
| 직무역량 | 문제해결 사례 | 정량 성과 | 기술 스택 나열 |
| 협업/갈등 | 상황-행동-결과 | 팀 규모/역할 | 자기 PR 위주 |

---

## 3. KB 근거 사용 규칙

### 사용 위치 제한

- **자기소개**: 대표 경험 1~2개만
- **지원동기**: 회사 선택과 연결되는 경험만
- **직무역량**: 문제 해결 사례 중심

### 사용 방식

```
❌ "CVPR 2025에 논문이 채택되었고, SOTA 대비 15% 성능 향상을 달성했습니다."
✅ "새로운 방법론을 제안하여 기존 한계를 극복했고, 이 과정에서 문제를 끝까지 파고드는 습관이 생겼습니다."
```

- 기술/성과는 **"증거"**이지 **"주인공"**이 아님
- 수치는 맥락 없이 나열하지 않음
- 경험이 "나"에게 어떤 영향을 주었는지 서술

---

## 4. 톤 & 문체 가이드

### 지양

- 과도한 자기 PR
- 논문/학회 이름 나열
- "~했습니다" 반복
- 추상적 표현 ("열정적으로", "최선을 다해")

### 지향

- 구체적 상황 묘사
- 사고 과정 공유
- 솔직하고 담백한 어조
- 질문에 대한 직접적 답변

---

## 5. Agent별 역할

### Planner
- 질문 유형 분류
- 합격 자소서 구조 적용
- KB 근거 사용 위치 지정

### Writer
- 사람 중심 서술
- 성향 → 경험 → 연결 흐름
- KB 근거는 지정된 위치에서만 사용

### Reviewer
- 성과 나열 여부 점검
- 질문 주제 집중도 검증
- "연구 보고서" 느낌 경고

### Integrator
- 최종 톤 조정
- 흐름 자연스럽게 통합
- 글자수 준수

---

## 6. 평가 기준 (CTS Scorer)

| 항목 | 가중치 | 설명 |
|-----|--------|------|
| 질문 집중도 | 0.25 | 질문 주제에 직접 답변하는가 |
| 사람 중심 | 0.25 | 성향/가치관이 드러나는가 |
| 구조 적합성 | 0.20 | 합격 자소서 구조를 따르는가 |
| 근거 적절성 | 0.15 | KB 근거가 적절한 위치에 사용되는가 |
| 회사 연결 | 0.15 | 회사/직무와 자연스럽게 연결되는가 |

---

## 7. 최종 점검 체크리스트

- [ ] 첫 문장이 "나는 ~한 사람입니다" 류의 성향 제시로 시작하는가?
- [ ] 논문/성과가 3개 이상 나열되지 않았는가?
- [ ] 수치가 맥락 없이 등장하지 않는가?
- [ ] 질문에 대한 답변이 명확한가?
- [ ] "연구 보고서"가 아닌 "자기소개서"로 읽히는가?
- [ ] 회사/직무 연결이 자연스러운가?

---

## 8. 자기소개서 생성 규칙 (제출용 완성 본문)

### 0) 목적
- 입력(회사/직무/문항/글자수/경험)을 바탕으로 **질문에 직접 답하는 제출용 본문**을 생성한다.
- 문체는 **존댓말(합니다/입니다)**로 고정한다.
- 글자수는 `application.target_char_min` ~ `application.target_char_max`를 **엄격히 준수**한다.
- 총점이 **9.5 이하이면 통과 불가**이며, 모듈별 기준 미달 시 해당 모듈만 반복 개선한다.

### 1) 공통 핵심 원칙 7가지
1. **질문 직접성**: 첫 문장에서 질문 의도에 바로 답한다.
2. **논리 흐름**: 성향/관심 → 경험 → 배운 점 → 직무 연결의 흐름을 유지한다.
3. **구체성**: 추상적 형용사 대신 행동·상황·결과로 증명한다.
4. **직무 연관성**: 경험을 직무 역량으로 해석한다.
5. **간결하고 일관된 문체**: 단문 위주, "합니다/입니다"로 통일한다.
6. **형식 표기 금지**: 소제목/번호/STAR/불릿/메타 문구를 사용하지 않는다.
7. **구체적 행동 강조**: “무엇”보다 “어떻게”에 집중한다.

### 2) 결정적 규칙(우선순위)
- **형용사 배제 및 수치화**: 추상 수식어 단독 사용 금지.
- **1인칭 주어 지양**: “저는/나는” 남발 금지, 가능한 완전 제거.
- **경험의 재해석**: 작은 경험이라도 직무 역량으로 연결.
- **안내/메타 문구 금지**: “이 문단에서는/다음으로/아래는” 등 금지.
- **맞춤법·통신어 금지**: 오탈자/줄임말/반말 금지.

### 3) 입력 요구사항
- 회사명, 직무명, 문항, 글자수 범위
- 경험 소재(역할/행동/결과/도구)
- 선택: 회사/직무 키워드, 강조 역량 키워드

### 4) 생성 규칙(제출용 본문)
```
[제출용 본문]
- 소제목/번호/STAR/불릿/메타 문구 없이 본문만 출력
- 1~3 문단 허용 (줄바꿈만 사용)
- 질문에 직접 답하는 완성 문장만 출력
```

### 5) 리스크 규칙
- 제공되지 않은 수치/사실을 가공하지 않는다.
- 정보가 부족하면 **[내용 보강 필요]**로 표시한다.
